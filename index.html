RouteReel

A zero-dependency Python CLI that turns a CSV of delivery stops into an efficient driving route — the order to visit them, mileage between each, and the total for the run.

Source on GitHub Back to portfolio
Overview

What it does

RouteReel solves the everyday dispatch problem: you have a truck, a depot, and a dozen drops — what's the shortest sensible way to hit them all and get back? Point it at a CSV of stops, name your depot, and it returns the visiting order, the mileage between each stop, and the total distance for the run. Zero external libraries, and it runs instantly.

Under the hood it's a small Traveling Salesman solver: real great-circle distances via the haversine formula, a nearest-neighbor heuristic to build the first route, then a 2-opt pass that un-crosses the path to shorten it — close to optimal for the stop counts one driver handles in a day.

Capabilities

Highlights

01

Real-world distance

Haversine great-circle mileage from latitude/longitude — not a flat grid approximation.

02

2-opt optimization

Nearest-neighbor builds a first route; a 2-opt pass un-crosses it to cut total distance.

03

Zero dependencies

Pure Python standard library. Clone and run — nothing to install, nothing to break.

04

Depot & one-way modes

Start from a named depot and return to it, or run one-way with --one-way.

Usage

Getting started

# clone and run
git clone https://github.com/karolisb123/routereel.git
cd routereel
python3 -m routereel.cli plan sample_data/stops.csv

# or install it as a command
pip install -e .
routereel plan stops.csv -d "Bolingbrook Depot"

# one-way run that doesn't return to the depot
routereel plan stops.csv -d "Bolingbrook Depot" --one-way
